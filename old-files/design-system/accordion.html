Accordion

Accordions let users expand and collapse sections of content. They reduce visual clutter by hiding content until it is needed, keeping pages scannable and organized.

Overview

A live, interactive accordion in single-open mode. Click any trigger to expand or collapse its panel.

Interactive demo — single open mode
Pulse is a comprehensive design system that provides reusable UI components, design tokens, and patterns for building consistent product experiences. It covers foundations like colors, typography, and spacing, as well as interactive components.
You can install Pulse via npm: npm install @pulse/components. Import individual components or use the full bundle. All components are tree-shakable and work with React, Vue, and plain HTML.
All Pulse components are designed and tested to meet WCAG 2.1 Level AA. This includes sufficient color contrast ratios, full keyboard navigability, proper ARIA roles and attributes, and screen reader support via semantic HTML.

Open Mode

Accordions support two open modes: single (only one panel open at a time) and multiple (any number of panels can be open simultaneously).

Sizes

Three sizes control the trigger row height. The default size is md (48px).

sm — 40px trigger
Compact 40px trigger row. Ideal for dense UIs such as sidebars or inspector panels.
md — 48px trigger (default)
Default 48px trigger row. The standard choice for most page layouts and content areas.
lg — 56px trigger
Large 56px trigger row. Best for marketing pages, landing pages, or FAQ sections with prominent headings.

With Icons

A leading icon can reinforce the topic of each item. The trailing chevron is always present and rotates 180° when open.

Leading icon + trailing chevron
Icons help users scan and identify sections quickly. Use consistent, recognizable icons that directly relate to the section content.
The leading icon sits inside .accordion-trigger-content before the label text. When the trigger is open, the icon inherits the secondary-600 color via CSS.
Keep icons at 18×18px for the default size. Scale proportionally for sm and lg variants — 16px for sm, 20px for lg.

With Badge / Tag

Badges or tags inside the trigger communicate status, counts, or labels without requiring the user to open the panel.

Tags communicate status at a glance. The badge sits inside the trigger content area and inherits the open-state color.
Use semantic badge colors to communicate status: green for active/success, yellow for warnings, purple for required or new items.
Counts in badges give users an immediate sense of how much is inside without opening the panel.

Do's & Don'ts

Do

Use accordions when content is logically grouped and users are likely to need only one or two sections at a time.

Don't

Don't use accordions to hide content users will always need. If all panels are frequently opened, use visible sections instead.

Do

Keep trigger labels short, descriptive, and action-oriented so users can predict the content before expanding.

Don't

Don't nest accordions more than one level deep. Multiple levels of collapsible content are difficult to navigate, especially with a keyboard.

Do

Use single-open mode by default for FAQs and navigation. Switch to multiple mode only when users need to compare content across sections.

Don't

Don't put critical form fields inside accordion panels — users may miss required fields that are collapsed when submitting.

Do

Always animate the panel open/close with a smooth max-height transition (0.25s ease) to give users clear feedback that content has expanded.

Don't

Don't use very long trigger labels that wrap to multiple lines — the trigger should remain a single line for visual clarity.

Accessibility

The accordion follows the WAI-ARIA Accordion pattern. All keyboard navigation and ARIA attributes are required for WCAG 2.1 AA compliance.

ARIA Roles & Attributes

  • Trigger: role="button" (implicit via <button>)
  • Trigger: aria-expanded="true|false"
  • Trigger: aria-controls="[panel-id]"
  • Panel: role="region"
  • Panel: aria-labelledby="[trigger-id]"
  • Disabled: aria-disabled="true" + HTML disabled

Keyboard Navigation

  • Tab — moves focus to next focusable element
  • Enter / Space — toggles the focused accordion item
  • ↑ Arrow Up — moves focus to previous trigger
  • ↓ Arrow Down — moves focus to next trigger
  • Home — moves focus to first trigger
  • End — moves focus to last trigger

Focus Management

  • Focus remains on the trigger after toggling
  • Focus ring uses outline: 2px solid #7f56d9 with 2px offset
  • Disabled triggers are excluded from the tab order
  • Panel content is keyboard reachable when open

Screen Reader Support

  • State change announced via aria-expanded
  • Panel region labelled by trigger text via aria-labelledby
  • Hidden content is removed from the accessibility tree when collapsed
  • Badge text should be meaningful (avoid icon-only badges without labels)

accordion.html — basic structure 
<!-- Default accordion (single-open) -->
<div class="accordion" data-mode="single">

  <div class="accordion-item">
    <button
      class="accordion-trigger"
      id="item-1-trigger"
      aria-expanded="false"
      aria-controls="item-1-panel"
    >
      <span class="accordion-trigger-content">
        Section title
      </span>
      <svg class="accordion-icon" aria-hidden="true">...</svg>
    </button>
    <div
      class="accordion-content-wrap"
      id="item-1-panel"
      role="region"
      aria-labelledby="item-1-trigger"
    >
      <div class="accordion-content">
        Panel content goes here.
      </div>
    </div>
  </div>

</div>

accordion.js — toggle logic 
// Initialize all accordions
document.querySelectorAll('.accordion').forEach(accordion => {
  const mode = accordion.dataset.mode || 'single';
  const triggers = accordion.querySelectorAll('.accordion-trigger:not([disabled])');

  triggers.forEach(trigger => {
    trigger.addEventListener('click', () => {
      const isOpen = trigger.classList.contains('is-open');
      const panelId = trigger.getAttribute('aria-controls');
      const panel = document.getElementById(panelId);

      // In single mode, close all others first
      if (mode === 'single' && !isOpen) {
        triggers.forEach(t => closeItem(t, accordion));
      }
      isOpen ? closeItem(trigger, accordion) : openItem(trigger, panel);
    });
  });
});

function openItem(trigger, panel) {
  trigger.classList.add('is-open');
  trigger.setAttribute('aria-expanded', 'true');
  panel.classList.add('is-open');
}

function closeItem(trigger, accordion) {
  const panelId = trigger.getAttribute('aria-controls');
  const panel = accordion.querySelector(`#${panelId}`);
  trigger.classList.remove('is-open');
  trigger.setAttribute('aria-expanded', 'false');
  if (panel) panel.classList.remove('is-open');
}
