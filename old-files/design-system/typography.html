TYPOGRAPHY

Typography

Pulse uses Inter as the primary typeface across all UI, with Poppins reserved for the H6 level. The scale is built on a 14px base with a 1.4 line-height ratio.

Stable Figma Styles

Font Family

Two typefaces are in use. Inter is used for all UI text. Poppins is used exclusively at the H6 level for decorative subheadings.

Do's & Don'ts

Do

Use the named type tokens consistently. Always pick from the scale — never invent a one-off font size like 13px or 17px.

Section heading

Body copy follows naturally here with proper hierarchy and line-height.
Don't

Don't skip more than one level in the heading hierarchy. Jumping from H1 to H4 confuses screen readers and breaks visual rhythm.

H1 heading here

← then jumping straight to body — no H2/H3
Do

Use text-sm or text-xs for labels, badges, timestamps and supporting captions — keep body copy at text-base.

Don't

Don't use font-weight: bold or font-weight: 600 directly. Use the bold-variant token (text-base-bold) which has the correct tracking and color pairing.

For Designers

TaskGuideline
Apply a text styleUse Text Styles panel in Figma — all tokens are pre-defined. Never apply manual font-size overrides.
Choosing heading levelUse H1 for page titles only. H2 for major section titles. H3–H5 for nested content. H6 for decorative UI labels (cards, tabs).
Responsive sizingOn mobile, reduce H1 to H2 size, H2 to H3. Body and small scales stay fixed.
Line lengthKeep body text between 60–80 characters per line. Use max-width: 65ch as a guide.

For Developers

ScenarioRecommended approach
Page headingsUse semantic HTML <h1>–<h6> tags, then apply the matching CSS class for visual styling
Body copySet font: 400 14px/1.4 'Inter', sans-serif on <body> and inherit — don't re-declare on every element
TruncationUse overflow: hidden; text-overflow: ellipsis; white-space: nowrap for single-line; -webkit-line-clamp for multiline
Responsive typeUse CSS clamp() for fluid headings: font-size: clamp(24px, 3vw, 35px)
