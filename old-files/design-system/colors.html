Do's & Don'ts

Do

Use semantic tokens (--text-primary, --color-error) in component code so colors update automatically when the theme changes.

Don't

Don't hard-code raw hex values like #29294c directly in component styles. Use the CSS variable equivalent instead.

Do

Test color pairings with WCAG contrast checkers before shipping. Aim for at least 4.5:1 for normal text, 3:1 for large text and UI components.

Don't

Don't use color alone to convey meaning. Always pair color with an icon, label, or pattern for users with color-vision deficiencies.

Do

Use the tint (50–100 step) for backgrounds and the 600–800 step for text on those backgrounds — keeping the same hue at both steps.

Don't

Don't mix semantic colors arbitrarily — e.g. a success-green icon inside an error-red card. Keep the semantic intent consistent within a component.

For Designers

TaskWhere to find it
Apply a color to an elementUse the Local variables panel → Color Tokens group. Never pick from the "Document colors" raw swatches.
Check contrast ratioSelect text layer → right panel → Contrast Ratio badge (with Figma's built-in checker or the Contrast plugin).
Add a new colorPropose it in the #design-system Slack channel first — new tokens require team sign-off before being added to the library.
Dark modeVariable modes are defined in the library file. Switch via the right panel → Layer → click the mode toggle.

For Developers

ScenarioRecommended approach
Text colorcolor: var(--text-primary) for headings, var(--text-secondary) for body, var(--text-tertiary) for captions
Backgroundbackground: var(--bg-surface) for cards, var(--bg-page) for layouts, var(--bg-subtle) for hover
Bordersborder: 1px solid var(--border-default) for most cases; var(--border-focus) on focus ring
Semantic statesUse var(--color-success), var(--color-error) etc. for icon colors; pair with var(--color-success-bg) for the container
Raw palette accessUse CSS custom properties defined at :root e.g. var(--secondary-600) — never hardcode hex values
