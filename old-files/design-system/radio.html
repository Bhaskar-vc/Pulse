RADIO

Radio

Radios allow users to select a single option from a list of mutually exclusive options. Supports two sizes, label, subtext, disabled state, and full keyboard accessibility.

Stable WCAG AA Figma · 745-29926

Basic

A radio group — all buttons share the same name. Selecting one automatically deselects the others. Always pre-select a sensible default.

Do's & Don'ts

✓ Always pre-select a default

Radio groups should always have one option selected on load. An empty group creates ambiguity and increases error rates on form submit.

✕ Don't leave all options empty

Starting with nothing selected is ambiguous — users don't know if a choice is required and errors spike on submit.

✓ Use for mutually exclusive options

When exactly one choice is valid and all options should be visible at once, radio buttons communicate that constraint clearly.

✕ Don't exceed 6 options

More than 6 radios should use a Select dropdown. Long lists require scrolling and make comparisons hard.

✓ Wrap groups in fieldset + legend

Gives screen readers the group context, so users understand what question the radio group answers.

✕ Don't mix sizes in one group

Mixing sm and md creates misaligned tap targets and visual inconsistency within the same group.

Accessibility

The native input[type=radio] is visually hidden but remains in the accessibility tree, providing correct ARIA role, keyboard behaviour, and form semantics automatically.

ARIA & Semantics

  • Native input[type=radio] provides role="radio" automatically
  • Wrap groups in <fieldset> + <legend> for group context
  • Custom visual is aria-hidden="true" — native input carries all semantics
  • Use ariaLabel or ariaLabelledBy when no visible label is present

Keyboard Interaction

  • Tab — enters the group (focuses selected item)
  • ↑ / ↓ or ← / → — navigate between options and select
  • Tab — exits the group to the next focusable element
  • Focus ring uses box-shadow — visible inside overflow:hidden

Color & Contrast

  • Unselected border #d0d5dd on white meets WCAG AA 3:1 non-text contrast
  • Selected fill #7f56d9 on white inner dot passes 4.5:1
  • Label #29294c on white: contrast ratio ≥ 7:1
  • Subtext #707087 on white: meets 4.5:1 for small text

Touch & Target Size

  • Minimum touch target is 44×44px — the <label> wrapper expands the hit area
  • Always include label text to maximise tappable area
  • Maintain at least 8px vertical gap between group items
  • Prefer md size on mobile — sm only on desktop

Implementation Notes

For Designers

  • Figma node: 745-29926
  • Use Size prop (sm / md) — never resize the circle frame manually
  • subtext slot is optional — detach and remove if unused
  • Maintain 8px vertical gap between group items
  • Use auto-layout column for groups — avoid manual positioning
  • Never override token colors directly on the component instance

For Developers

  • Always use native <input type="radio"> — hidden visually, not from the a11y tree
  • The inner dot is pure CSS: box-shadow: inset 0 0 0 4px #f9f5ff (sm) / 5px (md)
  • Group with shared name attribute — browser enforces mutual exclusion natively
  • Wrap groups in <fieldset> / <legend> for screen reader context
  • Disabled: add .is-disabled to wrapper AND disabled to native input
  • Angular: use [(ngModel)] with a shared model variable across the group
