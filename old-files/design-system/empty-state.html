EMPTY STATE

Empty State

Empty states communicate that no content is available — due to no data, a failed search, or a first-time experience. They guide users with a clear illustration, message, and optional action.

Stable Accessible Figma Ready

Overview

The default empty state uses a neutral illustrated box, a short title, and optional description. It matches the Figma "Empty/Basic" component exactly.

No Data

Variants

Four contextual illustrations cover the most common empty state scenarios: no data, no search results, no notifications, and no connection.

No Data There's nothing here yet. Add your first item to get started.
No Results Found We couldn't find anything matching your search. Try different keywords.
All Caught Up You have no new notifications. Check back later.
No Connection Check your internet connection and try again.
No Files Yet Upload your first file to this folder.
Page Not Found The page you're looking for doesn't exist or has been moved.

Accessibility

Illustration

  • Decorative SVGs should have aria-hidden="true"
  • Never use the illustration alone to convey meaning — always pair with text

Region

  • Wrap in role="status" or aria-live="polite" when the empty state appears dynamically after a user action (e.g., clearing filters)

Heading hierarchy

  • Use a semantic heading tag if the empty state is the primary content of the page: <h1> for full-page, <h2> / <h3> inside panels

Focus

  • When the empty state replaces content after a filter or search action, move focus to the empty state container or its heading so screen reader users are aware of the change
