Modal

Modals focus user attention on a task or critical information by overlaying the page with a dialog. Use them for confirmations, forms, alerts, and content that requires user action before continuing.

Padding 24px
Border radius 12px
Button height 36px
Backdrop rgba(52,64,84,0.7) + blur

Usage guidelines

✓ Do

Use modals for actions that require user confirmation or brief data entry. Keep content concise and give users a clear way to dismiss. Always include a close button and allow Esc to dismiss.

✕ Don't

Don't open a modal from inside another modal (avoid modal chains). Don't use modals for complex workflows or large amounts of content. Don't remove the ability to dismiss the modal.

✓ Do

Use intent-specific icon colors: green for success, amber for warnings, red for destructive actions. Match the primary button color to the intent.

✕ Don't

Don't use modals for non-critical information that could appear inline. Don't trigger modals without user intent (e.g. on page load). Keep modal titles short — one line is ideal.

Accessibility

Modals follow the ARIA dialog pattern and require specific focus management behaviors.

Focus management

  • When the modal opens, focus moves to the first focusable element inside
  • Tab cycles through focusable elements within the modal only (focus trap)
  • When the modal closes, focus returns to the trigger that opened it

Keyboard interaction

  • Tab / Shift+Tab — cycle focusable elements inside modal
  • Esc — closes the modal
  • Enter / Space — activates focused button

ARIA requirements

  • Modal element needs role="dialog" and aria-modal="true"
  • Use aria-labelledby pointing to the title
  • Use aria-describedby for the body text
  • Close button needs aria-label="Close"

Background scroll lock

  • Set document.body.style.overflow = 'hidden' when open
  • Restore overflow on close
  • Screen reader users should not be able to reach content behind the overlay

Code

HTML structure for the default modal with overlay and focus trap.

modal.html 
<!-- Modal overlay -->
<div class="modal-overlay"
     id="modal-1"
     role="dialog"
     aria-modal="true"
     aria-labelledby="modal-1-title">

  <div class="modal">
    <div class="modal-content">
      <div class="modal-icon-row">
        <div class="modal-feat-icon modal-feat-icon--success">
          <!-- icon svg -->
        </div>
        <button class="modal-close" aria-label="Close">...</button>
      </div>
      <div class="modal-text">
        <p class="modal-title" id="modal-1-title">Title</p>
        <p class="modal-body">Description text.</p>
      </div>
    </div>
    <div class="modal-actions modal-actions--right">
      <div class="modal-btn-group">
        <button class="modal-btn modal-btn--secondary">Cancel</button>
        <button class="modal-btn modal-btn--primary">Confirm</button>
      </div>
    </div>
  </div>

</div>
modal.js — open / close / focus trap 
function openModal(id) {
  const overlay = document.getElementById(id);
  overlay.classList.add('is-open');
  document.body.style.overflow = 'hidden';
  // Move focus to first focusable element
  const first = overlay.querySelector(
    'button, input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );
  if (first) first.focus();
}

function closeModal(id) {
  const overlay = document.getElementById(id);
  overlay.classList.remove('is-open');
  document.body.style.overflow = '';
}

// Close on backdrop click or Esc
document.querySelectorAll('.modal-overlay').forEach(overlay => {
  overlay.addEventListener('click', e => {
    if (e.target === overlay) closeModal(overlay.id);
  });
});
document.addEventListener('keydown', e => {
  if (e.key === 'Escape') {
    document.querySelectorAll('.modal-overlay.is-open')
      .forEach(o => closeModal(o.id));
  }
});
