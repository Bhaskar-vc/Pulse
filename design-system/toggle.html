Toggle

Toggles let users switch a setting immediately between on and off. Unlike checkboxes, they apply the change instantly — no submit action required. Use them for real-time preference changes, feature flags, and notification settings.

Stable WCAG AA Figma · 975-2864

Do's & Don'ts

✓ Use for instant setting changes

Toggles apply changes immediately without a submit button. This mental model is well-understood. Use them for settings that take effect in real time — dark mode, notifications, feature flags.

✕ Don't use in submit-based forms

If the user's preference only takes effect after clicking a "Save" button, use a checkbox instead. Toggles imply immediacy. Mixing them with form submission creates a confusing UX contract.

Receive emails (inside a form — use checkbox)
✓ Always provide a visible label

A toggle without a label gives no indication of what it controls. Even in compact settings rows, include a brief label or use aria-label at minimum for screen readers.

Notifications
✕ Don't use for mutually exclusive choices

Toggles are binary — on or off. If users need to pick one from several options (e.g., Monthly / Annual / Custom billing), use radio buttons. Toggles do not communicate mutual exclusivity.

✓ Use consistent sizes in a group

All toggles within a settings panel or list should be the same size. Consistency makes the UI predictable and easier to scan.

✕ Don't use toggles for destructive actions

If the "on" state triggers an irreversible or high-stakes action (e.g., "Delete account", "Wipe all data"), use a dedicated button with a confirmation dialog instead. Toggles imply reversibility.

Accessibility

The toggle uses role="switch" and aria-checked to communicate state to assistive technology. All interactive states meet WCAG 2.1 AA.

ARIA Roles & Attributes

  • Set role="switch" on the clickable element
  • Toggle aria-checked="true"/"false" on state change
  • Set aria-disabled="true" when disabled (don't just remove pointer-events)
  • Provide aria-label when no visible label is present

Keyboard Interaction

  • Tab — moves focus to the toggle
  • Space or Enter — activates the toggle
  • Focus ring uses purple-toned box-shadow so it works inside overflow containers
  • The native <button> element is used so keyboard handling is free

Color & Contrast

  • On state: #12b76a track — white thumb has sufficient contrast (4.5:1+)
  • Off state: rgba(0,0,0,.09) is intentionally subtle — pair with a label always
  • Label text (#29294c on white): contrast ratio ≥ 7:1
  • Supporting text (#707087): meets 4.5:1 for normal text

Motion & Animation

  • Thumb and track transitions are 0.2s ease — below the 400ms threshold for vestibular sensitivity
  • Respect prefers-reduced-motion by removing transitions in production
  • State change is also communicated via aria-checked — not just animation
  • Never use animation as the sole affordance for state change

Toggle JavaScript handler

/**
 * Toggle a switch on click.
 * Handles .is-on class, aria-checked, and optional onChange callback.
 */
function toggleSwitch(btn, onChange) {
  if (btn.classList.contains('is-disabled')) return;

  const isOn = btn.classList.toggle('is-on');
  btn.setAttribute('aria-checked', String(isOn));

  if (typeof onChange === 'function') {
    onChange(isOn);
  }
}

// Reduced-motion support
const mq = window.matchMedia('(prefers-reduced-motion: reduce)');
if (mq.matches) {
  document.querySelectorAll('.toggle-track, .toggle-thumb').forEach(el => {
    el.style.transition = 'none';
  });
}

Implementation Notes

For Designers

  • Figma component node: 975-2864
  • Use the Size prop: sm / md / lg
  • Use the State prop: Default / Disabled
  • On state uses the success-600 token (#12b76a) — do not swap for a different green
  • The thumb shadow is fixed — do not remove it in low-fidelity mocks
  • Track inset shadow gives depth — preserve in all contexts
  • Never use the toggle in a submit-form context — document this constraint in your handoff notes
  • Keep 16px minimum gap between the toggle and label in horizontal layouts

For Developers

  • Use <button role="switch"> — not a <div> or custom element
  • Always update aria-checked on every state change
  • Use aria-disabled="true" instead of the disabled attribute on <button> (disabled removes it from focus order)
  • Apply prefers-reduced-motion media query to remove CSS transitions
  • In React, drive state via useState — pass checked and onChange as controlled props
  • Travel formula: translateX(trackW - thumbD - 4px) where 4px = 2×2px inset
