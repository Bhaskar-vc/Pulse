Input

Input fields let users enter and edit text. They come in three types — Default, Input field (with leading icon), and Payment — each with full label, hint, validation, and state support.

Do's & Don'ts

Do

Always pair a visible <label> with every input. Use for/id to link them for screen readers.

Don't

Don't use placeholder text as a substitute for a label. Placeholders disappear when the user starts typing.

Do

Show inline error messages immediately below the field that caused them. Include the error icon alongside the text.

Enter a valid email address.
Don't

Don't only change border color to signal an error state. Always add a text message and icon — color-blind users will miss a color-only signal.

Do

Use the leading icon to reinforce the expected input type — email envelope, phone, lock, search — keeping icons semantic and consistent.

Don't

Don't use decorative or unrelated icons as leading icons. Avoid using star, sparkle, or abstract icons that don't reflect the input's purpose.

Error / Destructive state

<div class="field">
  <label class="field-label">Email</label>
  <div class="input-wrap state-error">
    <input type="email" aria-invalid="true" aria-describedby="email-err">
  </div>
  <span class="field-hint error" id="email-err" role="alert">
    Enter a valid email address.
  </span>
</div>

Payment / prefix-suffix

<div class="input-wrap">
  <div class="input-prefix">$</div>
  <input type="number" placeholder="0.00">
  <div class="input-suffix">USD</div>
</div>

Disabled

<div class="input-wrap state-disabled">
  <input type="text" placeholder="Not editable" disabled>
</div>

Accessibility

RequirementImplementation
Visible label Every input must have a visible <label> connected via for/id. Never use only aria-label or aria-placeholder.
Error association Add aria-invalid="true" to the <input> and aria-describedby="hint-id" pointing to the error message. Use role="alert" on the hint to announce it live.
Focus visible The purple ring (box-shadow: 0 0 0 3px rgba(127,86,217,.12)) provides a visible 3px focus indicator compliant with WCAG 2.4.11 Focus Appearance.
Disabled state Use the native disabled attribute so assistive technologies announce it. Don't use aria-disabled alone — it still allows focus.
Touch target The 40px height meets the WCAG 2.5.5 minimum 44×44px requirement when accounting for 2px padding on each side.
Placeholder contrast Placeholder color #bdbdc8 has a 2.5:1 contrast ratio — intentionally below 4.5:1, as placeholder is a hint not a label. Always provide a separate label.
Autocomplete Set autocomplete attributes (email, tel, current-password, etc.) on relevant fields to aid autofill and reduce friction.

For Designers

TaskGuideline
Choosing a typeUse Default for names, text, general fields. With leading icon for email, phone, search, URL. Payment for monetary values, codes, measurements.
Label copyKeep labels to 2–3 words. Sentence case. No colon. Describe the field — not the action (e.g. "Email address" not "Enter email").
Hint textUse for formatting requirements, constraints, or one-line context. Keep it under 80 chars. Don't repeat the label.
Error message copyBe specific: "Enter a valid email address" not just "Invalid email". Don't use ALL CAPS or exclamation marks.
WidthMatch input width to the expected content length — short for postal codes, full-width for addresses. Avoid inputs wider than their container.

For Developers

ScenarioRecommendation
Error validationValidate on blur (not on keystroke) for most fields. Validate on submit, then show inline errors. Clear errors as the user corrects them.
Password toggleToggle between type="password" and type="text" on the input element. Update the eye icon and its aria-label accordingly.
Controlled stateAlways manage both the value and the state-* class from a single source of truth. Don't let CSS and JS state diverge.
Character countPair with maxlength attribute. Update count on input event. Show as "N / max" right-aligned below the input.
Required fieldsUse native required attribute. Don't use asterisk (*) without an accessible legend explaining it.
