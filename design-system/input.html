All component properties mirroring the Figma component API.

Code

Copy-paste HTML for each input variant.

Default with label & hint

<div class="field">
  <label class="field-label" for="email">Email address</label>
  <div class="input-wrap">
    <!-- optional: leading icon -->
    <span class="input-icon">…svg…</span>
    <input type="email" id="email" placeholder="you@company.com">
  </div>
  <span class="field-hint">We'll never share your email.</span>
</div>

Error / Destructive state

<div class="input-wrap is-error">
  <span class="input-icon">…svg…</span>
  <input type="email" aria-invalid="true" aria-describedby="err">
  <span class="input-alert">…alert-circle svg…</span>
</div>
<span class="field-hint is-error" id="err" role="alert">
  Enter a valid email address.
</span>

Disabled

<div class="input-wrap is-disabled">
  <input type="text" placeholder="Not editable" disabled>
</div>

Payment / prefix-suffix

  <div class="input-prefix">USD</div>
  <input type="number" placeholder="0.00">

</div>

Do's & Don'ts

Do

Always pair a visible <label> with every input. Link them with for/id for screen readers.

Don't

Don't use placeholder text as a substitute for a label — it disappears when the user starts typing.

Do

Show inline error messages immediately below the field. Include an icon alongside the text for accessibility.

Enter a valid email address.
Don't

Don't rely only on border colour to signal an error. Always add a text message — colour-blind users will miss a colour-only signal.

Accessibility

RequirementImplementation
Visible label Every input must have a visible <label> linked via for/id. Never use aria-label alone.
Error association Add aria-invalid="true" to the input and aria-describedby pointing to the hint element. Add role="alert" on the hint to announce it live.
Focus visible The grey focus ring (0 0 0 4px #f2f4f7) renders outside the border — compliant with WCAG 2.4.11 Focus Appearance.
Disabled Use the native disabled attribute. The input is excluded from tab order automatically and background changes to #f2f4f7.
Colour contrast Filled text #29294c on white exceeds 7:1 (WCAG AAA). Label #707087 on white meets 4.5:1 (WCAG AA).
