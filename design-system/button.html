Button

Triggers an action or event. Available in 5 variants × 6 semantic colors × 3 sizes. Supports icons, loading states, and full keyboard accessibility.

Stable WCAG AA Figma node 8064-6597

Overview

Buttons communicate actions. Use one solid primary as the main call-to-action per view. Support variants for hierarchy and context — secondary, outlined, or light for alternatives; ghost for low-emphasis inline actions; tinted where subtle prominence is needed.

Do's and Don'ts

✓ Do
  • Use one solid primary button per section.
  • Use outlined or light for cancel/back alternatives.
  • Use error/solid for destructive, irreversible actions.
  • Keep labels short — 1–3 words, sentence case.
  • Always add aria-label on icon-only buttons.
  • Use <button> for actions, <a> for navigation.
✕ Don't
  • Don't use multiple solid primary buttons in one view.
  • Don't use div or span with click handlers instead of <button>.
  • Don't omit aria-label on icon-only buttons.
  • Don't use long labels or ALL CAPS text.
  • Don't mix conflicting semantic colors (e.g. error for save).
  • Don't use error style for non-destructive caution — use warning.
✓ Correct hierarchy
  • Primary → solid. Secondary → outlined or light.
✕ Wrong semantic
  • Don't use error color for safe, reversible actions.

Design guidelines
  • Use auto-layout — never detach button instances.
  • Set fix-height via token, not manual height override.
  • Minimum 8px gap between adjacent buttons.
  • On dark backgrounds use light / ghost variants, not solid.
  • Use disable-color state when button is loading (not plain disable).
  • Full-width buttons should only be used on mobile or inside modals.

For Developers

HTML semantics

  • Use <button> for all actions — never div, span.
  • Set type="button" or type="submit" explicitly.
  • Use <a href> only for actual page navigation.
  • Set disabled attribute when action unavailable.

CSS class naming

  • Base: btn
  • Variant: btn-solid-* btn-light-* btn-ghost-* btn-outlined-* btn-tinted-*
  • Size: btn-sm / btn-lg (md is default)
  • Colors: primary secondary success info warning error

States

  • Use CSS :hover, :focus-visible, :disabled.
  • Focus ring: box-shadow: 0 0 0 3px <ring-color>.
  • Never use outline: none without a visible :focus-visible alternative.
  • Loading: add spinner + aria-busy="true", set color to disable-color.

Tokens (CSS variables)

  • --fix-height — 32 / 36 / 40px
  • --h-padding — 12 / 16 / 20px
  • --v-padding — 2 / 4 / 6px
  • --h-gap — 8px (icon ↔ label)
  • --radius — 8px
  • --font — 13 / 14 / 15px

Accessibility

⌨️ Keyboard

  • Tab — move focus to button
  • Enter / Space — activate
  • Never set tabindex="-1" on interactive buttons
  • Always show :focus-visible ring

🖥️ Screen reader

  • Use clear, action-oriented labels ("Save", "Delete survey")
  • Icon-only: always set aria-label
  • Decorative icons: set aria-hidden="true"
  • Disabled state is announced automatically

🎨 Color contrast

  • Solid: white text on semantic fill — all meet WCAG AA
  • Light / ghost / outlined / tinted: text uses semantic dark token
  • Disabled: text-disabled on bg-disabled — intentionally lower contrast

📱 Touch target

  • Minimum 44×44px tap target (WCAG 2.5.5)
  • sm size (32px height) — use only on desktop or with extra padding
  • Icon-only: always at least 36×36px visible + 44×44 touch
