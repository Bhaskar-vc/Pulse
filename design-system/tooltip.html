TOOLTIP

Tooltip

A small informational popup that appears on hover or keyboard focus. Use tooltips to label icon-only buttons, clarify form field hints, and expose truncated text — never to deliver critical information or interactive content.

Stable Figma · 230-4910

Overview

Tooltips surface brief, supplementary information on demand. They appear after a short hover delay (300 ms by default) and disappear when the pointer leaves or focus moves away. Two visual variants — dark and light — adapt to different surface contexts. The component is built entirely with CSS hover and focus-within, with no JavaScript required for basic usage.

Usage

Tooltips are supplementary — the UI must be fully operable without them. They are not a substitute for proper labels.

When to use

  • Icon-only buttons that lack a visible text label
  • Form field hints that would clutter the layout if always visible
  • Truncated text — expose the full string on hover
  • Keyboard shortcut reminders (e.g. "Save — ⌘S")
  • Abbreviations or technical terms needing plain-language glosses

When NOT to use

  • Long or structured content — use a popover instead
  • Critical actions or error messages — must always be visible
  • Mobile-only experiences — tooltips are unreachable on touch
  • Information already visible nearby
  • Interactive elements (links, buttons) inside the tooltip

Accessibility

Tooltips must be reachable by keyboard and announced correctly to screen readers — not only available to mouse users.

ARIA pattern

  • Add role="tooltip" to the bubble element
  • Link trigger to bubble via aria-describedby
  • Tooltip text should not duplicate the trigger's accessible name
  • Never place focusable elements inside the tooltip

Keyboard & focus

  • Tooltip shows on Tab focus (CSS :focus-within)
  • Dismiss on Escape when JS-enhanced
  • The trigger element itself must be keyboard-focusable
  • Recommended hover delay: 300 ms to avoid accidental triggers

Motion

  • Use prefers-reduced-motion to disable translate animation
  • Keep animation subtle — opacity fade + 4 px translate only
  • Never use bouncy or large-scale animations for tooltips

Color contrast

  • Dark variant: white on #1d2939 — passes WCAG AA at all sizes
  • Light variant: #29294c on #fff — passes WCAG AA at all sizes
  • Never convey meaning through color alone inside a tooltip

Props

All configurable properties for the Tooltip component.

Prop Type Default Description
content string — The text displayed inside the tooltip. Required.
position "top" | "right" | "bottom" | "left" "top" Which side of the trigger element the tooltip appears on.
variant "dark" | "light" "dark" Visual style. Use dark on light surfaces, light on dark backgrounds.
delay number 300 Milliseconds before the tooltip appears after hover begins. Requires JS implementation.
maxWidth string "240px" Maximum width of the tooltip bubble. Increase for multi-line content.
disabled boolean false When true, the tooltip will not be shown regardless of hover or focus.
