STEPS

Steps / Progress

Step indicators guide users through multi-step workflows — onboarding, checkout, setup wizards. Each step shows completion state: done, active, or upcoming.

Stable Accessible Figma Ready

Overview

The default steps component uses the check-pill style — compact green pills connected by a dark line. Completed steps show a green checkmark, the active step has a green ring, and upcoming steps are grey.

CSS Classes

ClassDescription
.stepsHorizontal flex container — distributes steps evenly
.steps--sm28px nodes, smaller text
.steps--lg44px nodes, larger text
.steps--dot12px dot nodes — minimal progress indicator
.steps--verticalStacks steps in a column with a vertical connector line
.stepIndividual step — flex column, centered
.step.is-doneCompleted — purple filled node, purple connector to next
.step.is-activeCurrent step — purple ring, purple step title
.step.is-errorFailed step — red node with warning icon
.step-node36px circle — number or icon
.step-textLabel container below the node
.step-title13px / semibold step label
.step-desc12px muted supporting text

Code

steps.html
<div class="steps" role="list" aria-label="Checkout progress">

  <!-- Completed step -->
  <div class="step is-done" role="listitem">
    <div class="step-node"><!-- ✓ SVG --></div>
    <div class="step-text">
      <span class="step-title">Account</span>
      <span class="step-desc">Details saved</span>
    </div>
  </div>

  <!-- Active step -->
  <div class="step is-active" role="listitem" aria-current="step">
    <div class="step-node">2</div>
    <div class="step-text">
      <span class="step-title">Payment</span>
    </div>
  </div>

  <!-- Upcoming step -->
  <div class="step" role="listitem">
    <div class="step-node">3</div>
    <div class="step-text">
      <span class="step-title">Review</span>
    </div>
  </div>

</div>
