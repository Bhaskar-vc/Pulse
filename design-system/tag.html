Tag

Tags are compact labels used to categorize, filter, or describe content. They support semantic color variants, multiple styles, sizes, icons, and removable states for interactive use cases.

Stable WCAG AA Figma 230-4908

Overview

Tags communicate status, category, or metadata. The six semantic colors map directly to system states, and each supports four visual styles: solid, light, outlined, and tinted.

Variants

Four visual styles are available: Solid for prominent badges, Light for subtle inline labels, Outlined for low-visual-weight contexts, and Tinted — a combination of tinted background and semantic border for emphasis without full fill.

Interactive Removable Tags

Click the × on any tag to dismiss it. Use for selected filter chips, applied labels, or email recipients. Removed tags animate out and the area collapses gracefully.

Filter Tags

Filter tags toggle between selected and unselected states. Use them as a group to let users narrow results by category. Multiple can be selected simultaneously.

Props

Prop Type Default Description
color 'primary' | 'secondary' | 'success' | 'warning' | 'error' | 'info' 'primary' Semantic color variant. Applies background, text, and border token set.
variant 'solid' | 'light' | 'outlined' 'light' Visual style. Solid = filled bg; Light = tint bg; Outlined = border only.
size 'sm' | 'md' | 'lg' 'md' Tag height and font size. sm = 20px/12px, md = 24px/13px, lg = 28px/14px.
removable boolean false When true, renders a × dismiss button at the trailing end.
onRemove () => void — Callback fired when the × button is clicked. Required when removable is true.
icon ReactNode | 'dot' — Leading element. Pass 'dot' for a semantic dot, or a custom SVG node.
children string — The visible label text inside the tag. Keep short (1–3 words).

Do's & Don'ts

Do
  • Keep tag labels concise — 1 to 3 words maximum.
  • Use semantic colors consistently: green for active/success, red for errors, yellow for warnings.
  • Use the light style for inline labels within dense content like tables or cards.
  • Provide aria-label on the remove button ("Remove [label] tag").
  • Use outlined style when tags appear on colored or patterned backgrounds.
  • Group filter tags horizontally with consistent size and spacing.
  • Use status badges with a dot for records in a table.
Don't
  • Don't use long labels or wrap text — tags should always be single-line.
  • Don't mix solid and light variants in the same horizontal row without clear intent.
  • Don't use tags as primary call-to-action buttons — use the Button component instead.
  • Don't use more than 5–6 tags in a single line without a scroll or wrap strategy.
  • Don't rely solely on color to convey status — always include text.
  • Don't place removable tags in read-only contexts (e.g. inside view-only detail panels).
