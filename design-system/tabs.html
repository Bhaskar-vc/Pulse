Getting started with Pulse Tabs

Tabs are one of the most common navigation patterns in web applications. The Pulse Tab component supports three visual variants, three sizes, and optional icon and badge slots. All variants are fully keyboard navigable and screen-reader accessible.

Installation

Install via npm: npm install @pulse/tabs. The package ships as an ES module and CommonJS bundle. Import the component and its stylesheet, then mount it anywhere in your application.

Configuration options

Set variant to 'line'. Use size for 'sm', 'md', or 'lg'. Control the active tab via value prop or let it manage itself with defaultValue.

Do's & Don'ts

Do

Use tabs when the content groups are at the same hierarchical level and users are unlikely to need to compare content across panels.

Don't

Don't use tabs as a substitute for main navigation or breadcrumbs. Tabs are for secondary navigation within a single page context.

Do

Keep tab labels concise — 1 to 3 words. Append a count badge for tabs that contain lists or data sets.

Don't

Don't use more than 7 tabs in a single group. Beyond 7, consider restructuring content or using a side navigation pattern instead.

Do

Pre-select the most commonly used tab or the first tab by default. Never show an empty state as the default.

Don't

Don't nest tab groups inside tab panels. Two levels of tabs confuse orientation and break keyboard navigation expectations.

Accessibility

Tabs follow the WAI-ARIA Tabs pattern. The tablist, tab, and tabpanel roles, combined with ARIA attributes and keyboard handling, are required for WCAG 2.1 AA.

ARIA Roles & Attributes

  • Container: role="tablist" + aria-label
  • Tab button: role="tab"
  • Tab button: aria-selected="true|false"
  • Tab button: aria-controls="[panel-id]"
  • Panel: role="tabpanel"
  • Panel: aria-labelledby="[tab-id]"
  • Vertical: aria-orientation="vertical"

Keyboard Navigation

  • Tab — moves focus into and out of the tab group
  • ← / → Arrow — moves between horizontal tabs
  • ↑ / ↓ Arrow — moves between vertical tabs
  • Home — moves focus to the first tab
  • End — moves focus to the last tab
  • Enter / Space — activates the focused tab
  • Delete — closes a tab (if closeable)

Focus Management

  • Only the active tab is in the tab order (tabindex="0"). Other tabs use tabindex="-1".
  • When a tab is activated, focus moves to the corresponding tabpanel.
  • The tabpanel receives tabindex="0" to make it focusable.
  • When a tab is closed, focus moves to the preceding tab.

Color & Contrast

  • Active line tab: #6941c6 on #f9fafb — contrast 5.4:1 (AA pass)
  • Inactive tab: #707087 on #fff — contrast 4.6:1 (AA pass)
  • Count badge: ensure text contrast meets 4.5:1 minimum
  • Never rely on color alone to indicate the active state — the 2px border provides a second visual cue

tabs.js — keyboard-aware tab switcher 
function initTabs(tabsEl) {
  const tablist = tabsEl.querySelector('[role="tablist"]');
  const tabs = [...tabsEl.querySelectorAll('[role="tab"]')];
  const isVertical = tablist.getAttribute('aria-orientation') === 'vertical';

  tabs.forEach((tab, i) => {
    tab.addEventListener('click', () => activateTab(tab, tabs, tabsEl));

    tab.addEventListener('keydown', e => {
      const prev = isVertical ? 'ArrowUp'   : 'ArrowLeft';
      const next = isVertical ? 'ArrowDown'  : 'ArrowRight';
      if (e.key === prev) { e.preventDefault(); tabs[(i-1+tabs.length)%tabs.length].focus(); }
      if (e.key === next) { e.preventDefault(); tabs[(i+1)%tabs.length].focus(); }
      if (e.key === 'Home')  { e.preventDefault(); tabs[0].focus(); }
      if (e.key === 'End')   { e.preventDefault(); tabs[tabs.length-1].focus(); }
    });
  });
}

function activateTab(tab, tabs, tabsEl) {
  tabs.forEach(t => {
    t.classList.remove('is-active');
    t.setAttribute('aria-selected', 'false');
    const p = tabsEl.querySelector(`#${t.getAttribute('aria-controls')}`);
    if (p) p.classList.remove('is-active');
  });
  tab.classList.add('is-active');
  tab.setAttribute('aria-selected', 'true');
  const panel = tabsEl.querySelector(`#${tab.getAttribute('aria-controls')}`);
  if (panel) panel.classList.add('is-active');
}
